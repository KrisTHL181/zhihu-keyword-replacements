# 知乎关键词替换

Tampermonkey 用户脚本，在知乎页面中按作者/全局规则组自动替换指定关键词。

## 功能

- **规则组** — 支持创建多个规则组，每组可绑定特定作者；留空则为全局生效
- **作者匹配** — 自动检测页面内容的所属作者，仅应用匹配的规则组（支持部分名称匹配）
- **文本替换** — 在答案、文章、问题标题、评论、信息流等位置自动替换关键词
- **正则支持** — 每条规则可切换为正则表达式模式
- **动态监听** — MutationObserver 监听知乎 SPA 动态加载内容
- **持久化存储** — 规则、组、启用状态均通过 `GM_setValue` 持久化

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 浏览器扩展
2. 点击扩展图标 → **创建新脚本**（或从文件导入）
3. 将 [`zhihu-keyword-replacements.user.js`](./zhihu-keyword-replacements.user.js) 的全部内容粘贴进去
4. `Ctrl+S` 保存，脚本即生效

## 使用

1. 打开任意知乎页面（`www.zhihu.com` 或 `zhuanlan.zhihu.com`）
2. 点击右下角 **Kw** 浮动按钮，或 Tampermonkey 菜单 → **管理关键词替换规则**
3. 在弹出的面板中操作：

### 规则组管理

| 控件 | 说明 |
|------|------|
| 组下拉框 | 切换当前编辑的规则组 |
| `+` 按钮 | 新建规则组 |
| `-` 按钮 | 删除当前选中的组（至少保留一个） |

### 组设置

| 字段 | 说明 |
|------|------|
| 规则组名称 | 便于识别的名称 |
| 匹配作者 | 逗号分隔的作者名列表，支持部分匹配；**留空 = 全局规则**（对所有人有效） |

### 替换规则

| 字段 | 说明 |
|------|------|
| 查找关键词 | 要匹配的文本（或正则表达式） |
| 替换为 | 替换后的文本 |
| 正则 | 开启后"查找关键词"作为正则表达式解析 |
| 启用 | 控制单条规则是否生效 |

### 匹配逻辑

处理每个内容区块时：
1. 向上遍历 DOM 找到内容卡片容器（`.List-item` / `.AnswerCard` / `.ContentItem` 等）
2. 从卡片中提取作者名（`.AuthorInfo-name` 等）
3. **始终应用**全局规则组（authors 留空的组）
4. **额外应用**匹配到当前作者的规则组
5. 无法识别作者时仅应用全局规则
